# Results Gallery

End-to-end results on controlled synthetic data, with the operating ranges where
each backend is trustworthy.

!!! note "Figures"
    Rendered figures for this gallery are generated by a standalone script under the
    [`examples/`](https://github.com/JuliaSMLM/SMLMClustering.jl/tree/main/examples)
    project (heavy plotting/simulation dependencies are kept out of the package and
    the docs build). The benchmark tables below are reproduced from those runs.

## MRF density-regime accuracy

Characterized on a controlled 5×5 µm A431-mimic synthetic (kNN density estimator,
`k = 20`, 2 regimes, ~13–22 k emitters across 12 patches, aspect ratio 1–20).
Headline accuracy by density ratio (high / low):

| density ratio | kNN-MRF | voronoi-GMM (no MRF) |
|---------------|---------|----------------------|
| 1.2× | 35% | 65% |
| 1.5× | 69% | 67% |
| 2.0× | 89% | 72% |
| 3.0× | 95% | 69% |
| 5.0× | 96% | 87% |

**Use kNN-MRF when the density ratio is ≥ 2×.** Operational floor: ratio ≥ 1.65×
clears 75% accuracy; ratio ≥ 1.85× clears 85%. Below 2×, prefer calibrated soft
emissions (`calibrate_regime_gaussians` + `regime_gaussians`) when calibration ROIs
are available. See [MRF density-regime](@ref "MRF density-regime") for the method and
the configuration that produced these numbers.

## Patch-size and k-sensitivity

At fixed 2× density ratio, kNN-MRF holds 88–89% accuracy across 1.0–3.0 µm patches,
clears the 75% gate at 0.5 µm patches, and collapses below ~0.3 µm patches (the
resolution floor of the regime model). Over `density_k ∈ {5,…,80}` the accuracy
curve is concave with a broad plateau around `k = 15–20` (the default `k = 20` is
within rounding of the peak); low `k` loses precision to smoothing-amplified noise,
high `k` loses recall to density-mean regression.
